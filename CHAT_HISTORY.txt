Current implemented app behavior:
- Static web app organized as a Capacitor project for GitHub and iOS/TrollStore-style IPA workflows.
- Main web files are in www/index.html, www/styles.css, and www/script.js.
- GitHub Actions workflow lives at .github/workflows/main.yml.
- The devotional image is shown in a framed photo area from www/assets/radha-nila-madhava-button.png.
- Loading screen lasts 3 seconds.
- Main japa counter is a CSS mala/bead button.
- The number inside the bead button counts down from 108 to 0.
- Current bead, rounds, and remaining values update on each valid japa tap.
- State is saved locally in localStorage.
- Bead count and rounds reset each day.
- Pranams are required before each mala starts.
- The Pranams Done button is above the main japa bead button.
- If chanting is attempted before pranams are done, the app shows "Complete Pranam Mantras first." and opens Steps.
- That warning is layered above the full-screen Steps page so it remains visible during the screen change.
- Steps is now a full-screen page with a Back button.
- Steps includes the expanded pranam mantras and how-to-chant instructions.
- Offline malas can be added for malas completed on a real japa.
- Noon reminder notification is scheduled with the title "It's Time to Chant!" if a mala has not been completed by 12:00 PM.
- Next bead glow is kept; the moving bead marker was removed by request.
- The photo is back in its original position above the japa button.

Important build/package notes:
- This is a Capacitor project.
- The workflow can rebuild generated files in GitHub Actions.
- node_modules is intentionally not included to keep the project small.
- Run npm ci before Capacitor sync/build in a fresh environment.
- The user said they do not need signing and plans to use TrollStore.
- The app icon workflow expects a resources/icon.png file when the user adds it to Git.
